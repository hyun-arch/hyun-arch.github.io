## 0. 먼저, 지금 실태 (정직하게)

기획 전에 코드를 다 열어봤다. 사장님이 "재무제표·매출을 넣되 2중 비밀번호로 철저히 막아달라"고 하셨는데,
**지금 구조에서는 그 방식이 보안이 되지 않는다.** 이걸 먼저 짚지 않으면 사상누각이라 맨 앞에 둔다.

현재 아라미러의 보안 상태:

| 항목 | 실제 상태 | 의미 |
|------|-----------|------|
| Supabase anon 키 | `src/lib/config.js`에 **공개로 커밋됨** | 정상(공개용 키). 단, 실제 방어는 전부 RLS가 해야 함 |
| RLS 정책 | 모든 테이블(posts·carousels·videos·events) **"누구나 읽기+쓰기"** | 주소만 알면 **누구나 전부 열람·수정·삭제** 가능 |
| 어드민 비밀번호 | `admin.astro`에 `'arami'` **하드코딩**, sessionStorage로만 체크 | 소스 한 줄만 보면 뚫림. UI 가림막일 뿐 |
| 지도 잠금 | 2026-07-06에 **제거됨** (구조.md의 🔒는 옛 설명) | `src/data`의 모든 내용이 공개 사이트로 빌드됨 |
| DEV/PROD 분리 | **없음** (`import.meta.env.DEV` 게이팅 코드 0건) | "민감정보는 로컬에서만"이 실제로는 구현 안 됨 |

**결론: 지금 재무제표를 넣으면 그대로 전 세계에 공개된다.**
GitHub Pages는 정적 호스팅이라, 사이트에 실린 데이터는 브라우저가 이미 받아온 것 = 소스 보기/파일 다운로드로 누구나 열람 가능하다.
**정적 사이트에서 "클라이언트 비밀번호"는 잠긴 문이 아니라, 유리문에 붙인 커튼이다.** 2중이어도 마찬가지다.

그래서 v2의 1번 과제는 "예쁜 자물쇠 UI"가 아니라 **데이터가 물리적으로 어디에 사는가**를 먼저 정하는 것이다.

## 1. 보안·통제실 — 진짜 되는 3계층 구조

데이터를 민감도로 3층으로 나눈다. **핵심은 층마다 데이터가 사는 위치가 다르다는 것.**

| 계층 | 예시 | 어디에 사는가 | 실제 방어 | 비번의 역할 |
|------|------|--------------|-----------|-------------|
| 🟢 **공개** | 취미, 친구, 발행글, 스폰지클럽 | 배포 사이트(dist) | 없어도 됨 | 없음 |
| 🟡 **로그인 게이트** | 진행 중 프로젝트, 개인 메모 | 배포 사이트 + **Supabase Auth** | 서버 로그인(진짜) | 진짜 로그인 |
| 🔴 **비밀·히든** | **재무제표·매출·회사 기밀** | **배포에서 제외 · 내 PC(dev)에서만** | 내 노트북 자체 | 편의 잠금(얹기) |

세 층의 구현 방식:

- 🟢 **공개** — 지금처럼 그대로 빌드·배포. 변경 없음.
- 🟡 **로그인 게이트(선택)** — Supabase Auth를 붙이고, 해당 테이블 RLS를 `authenticated` 전용으로 바꾼다.
  그러면 배포 사이트에서도 **진짜 로그인 후에만** 열린다. anon(비로그인)은 아예 못 읽는다. → 이게 유일한 "온라인에서 진짜 잠긴" 방법. 개발 필요.
- 🔴 **비밀·히든** — 재무·매출은 **빌드 대상에서 물리적으로 제외**한다.
  - 방법: `src/data/_secret/` 폴더(`.gitignore` + 빌드 제외)에 두고, 이 데이터를 쓰는 페이지는 `import.meta.env.DEV`일 때만 렌더.
  - 결과: `npm run dev`(내 PC)에서는 다 보이고, **배포된 사이트에는 존재 자체가 없다.** → 유출 경로가 원천 차단.
  - 그 위에 사장님이 원하시는 **2중 비밀번호**를 "편의 잠금"으로 얹는다(내 PC에서 어깨너머 방지용). 단, 이건 어디까지나 로컬에서의 편의이지 온라인 보안이 아님을 명확히 한다.

> **통제실(`/hq`)이 이 3계층의 스위치보드가 된다.** 이미 `/hq`에 "🔴 사장님만 풀 수 있는 것 / 🟢 그냥 미는 것" 개념이 있으니, 이걸 데이터 공개/비공개 토글 + 잠금 상태 표시로 정식화한다.

**어드민 비번도 손본다:** 지금 `'arami'` 하드코딩은 없앤다. 로컬 편의 잠금은 `.env`의 값과 대조하는 방식으로, 온라인 보호가 필요한 건 Supabase Auth로. (하드코딩 문자열은 무조건 제거)

## 2. 데이터 통합 — 흩어진 것을 하나로

"나 = 시스템"이 되려면 흩어진 소스를 아라미러의 단일 모델로 모은다. 각 소스를 위 3계층에 배정한다.

| 소스 | 내용 | 계층 | 편입 방식 |
|------|------|------|-----------|
| 진행 중 세션·업무 (Typeless, claude.cmd, Opus 작업) | 지금 하는 일들 | 🟡/🟢 | `events` 로그 + 프로젝트 카드로 정규화 |
| 텔레그램 저널 | 던진 생각들 | 🟡 | 기존 저널 파이프라인 → 지도/게시판으로 승격 |
| 클로드 사용량 | 사용 통계 | 🟡 | 관제실 지표 카드 |
| **재무제표·매출** | **회사 기밀** | 🔴 | `_secret/` 로컬 전용. 배포 제외 |

**설계 원칙(기존 마스터 스펙 유지):** 데이터는 컴포넌트가 아니라 `src/data/`에 단일 소스로 둔다.
v2에서는 여기에 **계층 필드(`tier: 'public' | 'auth' | 'secret'`)**를 도입해, 빌드·표시·잠금이 전부 이 필드 하나를 보게 한다. (한 곳만 바꾸면 전 페이지 반영)

## 5. 에이전트 체계

"나에 대한 모든 것 + 회사 + 생각"이 한곳(아라미러)에 모이고, 그걸 돕는 에이전트를 3종으로 분류한다.
기존 "파일 = 에이전트" 개념(`/how`)과 CLAUDE.md 페르소나 방식 위에 얹는다.

| 유형 | 역할 | 이름(안) | 기반 |
|------|------|----------|------|
| **비서형** | 내가 말하는 걸 그대로 받아 정리·실행 | (예: "미러") | 텔레그램 → 저널 파이프라인 |
| **정보수집형** | 외부 아이디어를 크롤링·수집 | (예: "채집이") | 자동리서치/크롤 → 생각의 벽 외부칸 |
| **자문형** | 특정 인물의 철학·글로 나에게 **질문을 던지는** 에이전트 | **최진석 교수**, **고명환 작가** (각각 이름 부여) | 인물별 사상 요약 + 질문 프롬프트 |

> ⚠️ 참고: 지금 `/masters` 페이지는 **잡스·머스크**다. 사장님이 말씀하신 자문형(최진석·고명환)은 **다른 신규 인물**이라, 별도 자문 에이전트로 신설하는 방향이 맞다. (기존 masters와 통합할지 신설할지는 결정 안건)

## 8. 이번 주 슬라이스 #1 — 보안 3계층 뼈대 (실행 계획)

D2 결정에 따라, 데이터를 붓기 전에 **안전한 그릇**부터 만든다. 순서:

1. **`tier` 필드 도입** — `src/data`의 각 항목에 `tier: 'public' | 'auth' | 'secret'`. 빌드·표시·잠금이 전부 이 필드 하나를 본다.
2. **🔴 secret 빌드제외 파이프라인** — `src/data/_secret/`(gitignore + 빌드 제외) 신설. secret 데이터를 쓰는 렌더는 `import.meta.env.DEV`에서만. 배포 빌드엔 존재 자체가 없음(재무·매출 자리).
3. **하드코딩 비번 제거** — `admin.astro`의 `'arami'` 제거. 로컬 편의 잠금은 `.env` 값 대조로 교체.
4. **🟡 auth 스캐폴딩** — Supabase Auth 로그인 붙이고, auth-tier 테이블 RLS를 `authenticated` 전용으로 전환하는 구조 준비(실데이터 이관은 다음 슬라이스).
5. **통제실(`/hq`)에 잠금 상태 표시** — 각 영역이 🟢/🟡/🔴 어느 계층인지, 잠김/열림 상태를 한눈에.
6. **검증** — `npm run build` 초록 + 배포 산출물(dist)에 secret 문자열이 **하나도 없음**을 grep으로 확인.

> 완료 기준: 배포 dist를 아무리 뒤져도 재무·매출 흔적이 안 나오고, 준민감은 로그인 없이는 안 열린다.

- `src/data/_secret/` 신설 — 🔴 로컬 전용 비밀 데이터 폴더. `finance.example.js`(커밋용 템플릿) + `README.md` + 실제 `finance.js`(로컬 전용).
