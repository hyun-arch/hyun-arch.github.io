# 캐러셀 디자인 시스템 v1.0 — Aramirror

> "레퍼런스로 찾은 디자인을 디자인 시스템 MD로 저장" (사장님 방식).
> 이 문서 = 캐러셀 아웃풋의 **단일 기준(Single Source of Truth)**.
> AI에게 "이 디자인 시스템대로 만들어줘" 하면 매번 같은 결이 나오게 하는 규칙.
> 실제 구현: `src/pages/studio/slide/[n].astro`, 콘텐츠: `src/data/carousel.js`

## 1. 캔버스 규격 (Canvas Specs)
| 항목 | 값 |
|---|---|
| 비율 | 4:5 (Portrait) |
| 픽셀 | **1080 × 1350** |
| 여백(padding) | 상 92 / 좌우 84 / 하 84 px |
| 포맷 | PNG |

## 2. 컬러 (고정 — 변경 없이)
| 역할 | Hex |
|---|---|
| **포인트(브랜드)** | `#e9ed12` 노랑 — 단 하나의 색 |
| 다크 배경 | `#17140f` (표지는 라디얼 그라데이션) |
| 라이트 배경 | `#faf6ef` 크림 |
| 다크 본문 텍스트 | `#f4f0e6` / 서브 `#d3ccbb` |
| 라이트 본문 텍스트 | `#17140f` / 서브 `#5a5044` |

규칙: 색은 노랑 하나만. 강조는 **색이 아니라 굵기**로. 빨강·초록은 텍스트가 아니라 이모지(✅❌)로만.

## 3. 타이포그래피
- 본문/제목: **Pretendard**
- 소제목(kicker)·숫자: **Spline Sans Mono**
- 제목 h1: 88px (표지 108px), **ExtraBold 800**, 자간 -0.035em
- 서브: 34px / 500, 포인트 40px / 700
- 불릿: 34px / 600

## 4. 컴포넌트 (반복 요소)
- **kicker**: 모노 대문자 + 앞에 노랑 바(46×5px). 예 "치트키 ① · 쉽게 말하기"
- **강조 밴드(mark)**: 노랑 배경 + 검정 글자, radius 6px — 시그니처
- **고스트 숫자**: 우상단 배경에 큰 ①②③ (다크 노랑 14%, 라이트 먹 5%)
- **하단 워터마크**: ● @aramirror · Aramirror + 페이지 점(현재=노랑 바)

## 5. 슬라이드 구조 (템플릿)
1. **표지**: 후킹 카피 + 저장 유도(1장에서 이탈 방지) — 다크
2. **본문(치트키)**: 라벨 + 큰 제목(강조밴드) + 서브 + 불릿 3개, 다크/라이트 번갈아
3. **요약**: 전체 리스트 한 장 — 라이트
4. **마무리(CTA)**: 저장·팔로우·댓글→DM — 다크
- 슬라이드 수: 주제별 7~10장 (현재 8장)

## 6. 발행 (캡션·해시태그·마니챗)
- 캡션 구조: 후크 → 본문 → CTA → 해시태그 (플랫폼별 톤 다르게)
- 마니챗: 댓글 트리거 "치트키" → 오프닝 → 팔로우 확인 → 전달
- 상세: `src/data/carousel.js` 의 `captions`, `manychat`
