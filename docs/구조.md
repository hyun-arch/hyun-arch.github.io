## 데이터 모델
- `src/data/mirror.js` — 전체 지도의 **단일 콘텐츠 모델**.
  - `domains[]` → `branches[]` → `items[]` (각 항목: t/slug/to/status/lead/points/links)
  - `detailItems()` — `/m/[slug]`가 상세 페이지 생성할 항목만 추출
  - `statusMeta()` — 상태 라벨/색 (live/building/seed/idea)
- `src/scripts/store.js` — 게시판·페이지관리 데이터 레이어 (Supabase 또는 localStorage)

## 디자인 시스템
- `src/styles/global.css` — 토큰(색·폰트·radius) 한 곳. **여길 바꾸면 전 페이지 반영.**
  - 팔레트: 크림 `#faf6ef` / 코랄 `#c2603b` / 먹색 `#211c16`
  - `--mono` = Spline Sans Mono (소제목·숫자)
  - 재사용 유틸: `.btn`/`.btn-outline`, `.result`, `section.dark-section`+`.num-grid`
- `src/layouts/Base.astro` — 공통 레이아웃 (nav + 폰트 로드 + footer)

## 배포
- GitHub Pages, `gh-pages` 브랜치 (prebuilt dist + `.nojekyll`), Actions 없음.
- 현재 라이브(hyun-arch.github.io)는 **옛 디자인** — 새 디자인은 아직 로컬만.
