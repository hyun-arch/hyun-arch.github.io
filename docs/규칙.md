# 아람미러 통일 규칙 (Single Source of Truth)

> "도구가 무엇이든, 규칙이 먼저다." 클로드 챗·디자인·캐러셀 OS 무엇을 만들든
> 이 MD의 규칙을 먼저 따른다. 규칙이 통일돼야 매번 같은 결이 나온다.
> 관련 상세: [구조.md](./구조.md) · [캐러셀-디자인시스템.md](./캐러셀-디자인시스템.md)

## 1. 정체성
- 아람 = 아름다운 사람. **AI는 나를 대신하지 않는다 — 거울이 된다.**
- AI는 도구. **판단하고 결정하는 건 나(주체성).** 손은 AI가, 방향·발행은 사람이.

## 2. 디자인 시스템 (엄격 — 매번 동일)
| | 값 |
|---|---|
| 웹사이트 배경/포인트 | 크림 `#faf6ef` / 코랄 `#c2603b` / 먹색 `#211c16` |
| 인스타 캐러셀 포인트 | **노랑 `#e9ed12` (변경 없이)** — 마스코트: 밤티(노란 강아지) |
| 폰트 | Pretendard(본문) · Spline Sans Mono(소제목·숫자) |
| 강조 | 색이 아니라 굵기로. 강조어만 포인트색/밴드 |
| 소제목(kicker) | 모노스페이스 대문자 + 앞 포인트색 바 |

**뼈대는 공통, 목소리는 각자:** 사이즈·컬러 규칙·폰트·배치·그리드는 모두가 지키는 공통 뼈대(누가 만들어도 한 브랜드로 보이게). 그 위에서 캐릭터·개인 컬러로 개성만 다르게. (스폰지클럽식 — 각자 밤티 색이 다름)

## 3. 카피 톤앤매너
- 쉬운 우리말·짧은 문장·1인칭. 전문용어 쓰면 바로 풀어서.
- 결론 먼저(모닝 리포트: 제목 한 줄 + 핵심 3점 + So what).
- 근거 없는 수치는 지어내지 말고 `[확인 필요]`. 애매하면 되묻기.

## 4. 정보 구조
- **큰 구조 → 작은 구조** 드릴다운. 처음엔 큰 단위만, 클릭하면 펼쳐짐. 다 나열 금지.
- 만들기 전 **구조도(기획)를 먼저 그림으로** 보여주고 승인받기.

## 5. 작업 방식 (내가 이끌고, 클로드코드가 정리)
- 아이디어·방향은 사장님, 정리는 클로드코드가 **MD로**.
- 항상 ①구조화 → ②그 구조대로 폴더링 → ③칸마다 .md 파일.
- 텔레그램은 **답변부터 하고 시작**. 침묵 구간 금지. 결과는 스샷으로 시각화.

## 6. 데이터·보안
- 아카이빙 DB: **Supabase**(now) → **Obsidian**(later).
- 결과물은 저장(DB) 안 하면 날아감 → 만든 건 반드시 보관함에.
- API 키: `.env`만 · `.gitignore` 추가 · 값 절대 출력 금지.

## 7. 파이프라인 (생각 → 지식 → 자산)
- 1층 사고: 텔레그램 마찰0 캡처(**저장 먼저(raw), 정리는 나중**).
- 2층 지식: 밤마다 자동 연결·위키 승격. 내 것(자산) vs 외부(연료) 분리.
- 3층 자산: 글·발표·캐러셀·SNS로 발행. 원소스 멀티유즈.
