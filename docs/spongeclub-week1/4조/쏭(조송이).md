# 쏭(조송이) — Tally — 캡처+클로드코드 두뇌의 라이프 OS

- **조**: 4조
- **한 줄**: 텔레그램 raw 캡처(즉시 저장)+클로드코드 정리 → 궁극엔 로컬 모델로 자립

## 문제
생각을 폰에서 담을 곳이 없어 사라지고 정리는 metered API에 의존

## 아이디어
던지는 순간 raw 즉시 저장(durable-first) 절대 안 잃고, 정리 결과를 정답 코퍼스로 쌓아 로컬 모델 자립 3단계

## 태그
#캡처우선 #로컬자립 #라이프OS

## 아람미러 적용
데일리 캡처에 “저장 먼저, 정리는 나중” durable-first → 봇이 꺼져도 말이 유실 안 되는 raw 계층 분리
