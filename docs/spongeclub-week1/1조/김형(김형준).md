# 김형(김형준) — PARY OS 보완 + MSCW 우선순위 앱

- **조**: 1조
- **한 줄**: 회사 OS는 3,900줄인데 나를 위한 시간 규칙은 0줄 — 운영자 본인 규칙을 추가

## 문제
큰 프로젝트 끝나면 계획이 무너지고 자투리 시간을 흘려보냄

## 아이디어
“종료 후 24시간 내 재계획”·“이번 주 안 할 일(WON’T)” 두 규칙 명문화 + MUST/SHOULD/COULD/WON’T 앱

## 태그
#우선순위 #MSCW #운영규칙

## 아람미러 적용
아침 리포트에 WON’T(오늘 안 할 일) + 큰 일 끝난 뒤 재계획 트리거를 넣어 계획 붕괴를 시스템이 잡기
