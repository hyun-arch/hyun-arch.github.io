## 0. 개선 프로세스 (표준 루프)

모든 기능은 아래 6단계를 순서대로 밟는다. 이 문서가 ①②의 산출물이다.

1. **읽기(Read)** — 해당 페이지 + 데이터 + 공유 로직(store.js, backend.js, global.css)까지 함께 읽는다.
2. **기획(Spec)** — 현재 상태 / 디테일 갭 / 버그를 이 문서에 기록한다.
3. **우선순위(Rank)** — 보안 > 버그 > 데이터 정합성 > UX 디테일 > 확장. 키·외부서비스 불필요한 것부터.
4. **적용(Apply)** — 빌드가 항상 초록(green)을 유지하도록 작은 단위로 커밋.
5. **검증(Verify)** — `npm run build` 통과 + 헤드리스 스크린샷 자가검토(메모리 규칙: verify-visuals-before-reporting).
6. **보고(Report)** — 증거(파일·스샷)와 함께 사장님께 텔레그램 보고.

**설계 원칙(코드 전반에 적용):**
- 데이터는 컴포넌트가 아니라 `src/data/`에 둔다(단일 소스). 홈·목록·지도가 같은 배열을 공유.
- 키/시크릿은 `.env`. 정적 배포라 anon 키는 노출되므로 **보안은 RLS로**.
- 날짜는 항상 로컬(KST) 기준. `toISOString()`(UTC) 금지.
- 외부 CDN 폰트 의존 최소화(오프라인·캡처·클라우드 빌드 대비).
- 고아 페이지 금지 — 모든 페이지는 nav/footer/홈 중 한 곳에서 진입 가능해야 한다.

## 3. 이번 라운드 적용 내역 (실행 로그)

이 섹션은 실제 코드에 반영한 것만 기록한다(빌드 초록 유지).

- **[C1·C2] 홈/개발일지 단일 소스화**: `src/data/logs.js` 신설. 홈은 최신 3건 + "전체 기록 보기 →", 목록은 전체를 같은 배열에서 렌더.
- **[B1] 관리자 날짜 버그**: `admin.astro` UTC → 로컬 날짜(`localDate()`).
- **[B2] 몰입 데이터 정합성**: `flow.astro` — 기록별 삭제(×) 버튼, 12시간 초과 세션 저장 시 경고·확인, 초 단위 보관.
- **[C3] 캐러셀 색 정정**: 코랄 #c2603b → 노랑 #e9ed12 (문서-산출물 일치).
- **[C4] 생각의 벽 죽은 링크**: "준비 중" 배지 + 비활성 스타일 + `← 전체 지도` 백링크.
- **[O1] 고아 페이지 진입점**: footer에 `농구클럽(GTB)`·`스폰지클럽` 링크.
- **[O3] 고아 코드 삭제**: `src/data/mirror.enc.js`, `src/lib/mirror-crypto.js` 제거.

### 3-S1. 보안 결정 안건 (사장님 확인 필요)
게시판/캐러셀은 지금 **누구나 쓰기·삭제 가능**(anon 전면 허용). 두 갈래:
- (A) **현행 유지** — 개인·저트래픽 프로젝트라 실용 우선. 리스크: 주소를 아는 제3자가 글 삭제/변조 가능.
- (B) **쓰기 잠금** — RLS를 authenticated 전용으로. 단 로그인(Supabase Auth) 붙이기 전엔 게시판 쓰기가 멈춤. 추가 개발 필요.
→ 기본은 (A) 유지. 사장님이 (B) 원하면 다음 라운드에 Auth까지 붙여 완성.
