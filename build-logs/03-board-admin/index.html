#03 게시판 · 관리자

2026-07-04 · Build Log

재활용이 자동화의 핵심이고, 그것을 형태로 굳힌 것이 스킬이다. 게시판도 마찬가지 — 한 번 세워두면 그다음은 쭉 재활용된다.

어디서 왔나 — 이기적스폰지 공유회

2026-07-04 에이든의 공유회 아젠다는 다섯이었다.

  1. 기본 스킬
  2. KEY 연결 → CRUD 게시판
  3. 딥인터뷰 + 기본 스킬
  4. 씽킹노트(탐욕적으로 공부하기)
  5. 서브에이전트와 동적 워크플로우

두 번째가 곧장 아라미러에 필요한 것이었다. 공유회에선 .env에 KEY를 넣어 GitHub·Supabase·Vercel로 배선하고, 실제 어드민 콘솔(BASEBOARD — 대시보드·게시글·상태(공개)·처리(숨김/삭제))까지 만들었다. “한 번 연결이 힘들지, 그다음은 쭉 편해진다.”

무엇을 만들었나

아라미러는 GitHub Pages 정적 사이트다. 그래서 서버 DB 대신 브라우저 스토어(localStorage) + JSON 내보내기/가져오기로, 지금 당장 동작하는 CRUD를 먼저 세웠다.

  • 게시판 /board — 카테고리(생각·빌드로그·글·공유회·공지) 필터, 글 읽기. 공개된 글만 노출.
  • 관리자 /admin — 통계 대시보드, 글 생성·수정·삭제·게시/숨김, 웹페이지 상위 노출 관리, JSON 백업/복원, 암호 게이트.
  • 웹페이지 상위 관리 — 사이트 실제 라우트를 한 표에서 노출/숨김·메모로 관리.

데이터 레이어(src/scripts/store.js)는 UI와 분리했다. 이 경계 덕분에 나중에 저장소만 갈아끼우면 된다.

승격의 문은 열어둔다

지금은 로컬 스토어지만, 공유회 아젠다 ② 그대로 .env + Supabase + Vercel로 연결하면 이 콘솔이 그대로 다중 사용자 실서버 CRUD가 된다. 스토어 인터페이스(getPosts / createPost / updatePost / deletePost)는 이미 그 승격을 염두에 두고 잘라두었다.

남은 메모

  • 이 문서를 정리하려고 공유회 스크린샷 32장을 서브에이전트로 병렬 MECE 정리시켰다. 공유회 5번(서브에이전트/동적 워크플로우)을 문서화 과정에서 그대로 써먹은 셈이다.
  • 마크다운을 원천 포맷으로 — 게시판 본문도 렌더링에 기대지 않는 구조로 남긴다.

AI는 나를 대신하지 않는다. 게시판도 나를 대신 쓰지 않는다 — 내 기록이 자산이 되도록 비출 뿐이다.
