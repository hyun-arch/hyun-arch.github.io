# #05 v2 — 그릇부터 다시

<p class="meta">2026-07-19 · Build Log</p>

> 새 기능을 얹기 전에, 그걸 안전하게 담을 그릇부터. 오늘은 그 그릇을 만든 날이다.

## 발단 — "재무를 넣되 절대 새면 안 된다"

회사 재무·매출까지 OS에 넣어 "나 = 시스템"으로 만들고 싶었다. "비밀번호 2중으로 막아줘"라고 했는데, 정직한 답이 돌아왔다. **정적 사이트에서 클라이언트 비밀번호는 잠긴 문이 아니라 유리문에 친 커튼이다.** 화면에 뜬 데이터는 이미 브라우저가 받아온 것이라, 소스만 보면 다 보인다.

더 놀란 건 현재 상태였다. DB는 "누구나 읽기·쓰기", 관리자 비번은 소스에 `arami`로 박혀 있었고, 민감정보 잠금은 예전에 제거돼 있었다. **재무를 그냥 넣었으면 그대로 전 세계 공개될 뻔했다.**

## 무엇을 세웠나

- **보안 3계층** — 🟢 공개 / 🟡 로그인 / 🔴 비밀. 재무·매출은 🔴 로, **배포에서 물리적으로 제외**하고 내 PC(로컬)에서만 열리는 금고(`/vault`)로.
- **빌드 제외 파이프라인** — `import.meta.env.DEV` + `import.meta.glob` 으로, 프로덕션 빌드엔 비밀 데이터가 통째로 사라진다(트리셰이킹). 배포 산출물을 grep 해서 **재무 문자열 0건**을 확인하고서야 믿었다.
- **하드코딩 비번 제거** — `arami` 를 지우고 `.env` 기반 편의 잠금으로.

## 셸을 자동화하고, OS에 모으다

- **자동 셸 선물 봇 편입** — 매일 사라지는 셸을 조원에게 공정하게 순환 선물. **웹(클라우드)·PC 앱** 두 방식을 실제로 돌려보고, **셸 스튜디오(`/shell-studio`)**로 한 화면에 모았다. 남들도 쓰게 매뉴얼까지.
- **흩어진 프로그램을 지도에** — 매출 대시보드·재무재표·CRM·Claude 사용량·Calendar 등을 지도에 등록. 재무는 이름만 두고 숫자는 금고에만.
- **재무 실데이터 연동** — Revenue 매출 + 3개년 재무상태표(엑셀)를 금고에 실제로 연결.

## 오늘의 교훈

- **보안은 예쁜 자물쇠가 아니라, 데이터가 물리적으로 어디 사는가다.**
- **"됐다"는 실제로 눌러봐야 진짜 된 거다.** 정적 페이지 링크가 dev에서 404 나던 걸, 응답코드를 끝까지 찍어 잡았다.
- 편한 거짓말보다 불편한 사실. "그 방식으론 안 됩니다"가 진짜 도움이었다.
