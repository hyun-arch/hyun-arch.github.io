0. 무엇이 문제였나 (진단)

사장님이 링크를 열면 아무 변화가 없어 보였던 진짜 원인:

  • 배포 누락기획·수정은 로컬에 쌓여 있었지만 라이브(gh-pages)에 올리지 않았다 → 옛 버전만 보였다.
  • 관제실 분산hq · order · eval · admin 4개가 서로 떨어진 별도 페이지였다.
  • 기능 디테일 미완각 기능의 세부(검색·필터·프리셋·보관함 등)가 기획만 되고 미구현이었다.

3. 통합된 통제실 구조

아라미러 (hyun-arch.github.io)  ← 한 개의 웹, 하나의 네비
└─ /hq  관제실  ← 여기 하나로 다 확인·통제
   ├─ 상단: 아라미러 본체 네비(지도·벽·몰입·게시판·글·캐러셀·발행 …)
   └─ 탭:  오늘 · 프로젝트 · 핵심지표 · 자동화
           · 주문서(임베드) · 평가(임베드) · 관리자(임베드) · 빠른실행
결과 — /hq 하나만 열면: 오늘 할 일·프로젝트 현황·지표를 보고, 게시판/페이지를 관리(관리자 탭)하고, 주문서·평가를 채우고, 사이트 어느 화면으로든 바로 이동. 아라미러와 통제실이 한 페이지로 합쳐졌다.
